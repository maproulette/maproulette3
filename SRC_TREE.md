## src/api/

API hooks organized by domain. Each `index.ts` aggregates sub-modules into a namespace object.

- `index.ts` тАФ Top-level API namespace (`api.user`, `api.task`, `api.challenge`, etc.)
- `osm.ts` тАФ OpenStreetMap API (changesets, elements, Overpass)
- `project.ts` тАФ Project CRUD, listing, stats
- `search.ts` тАФ Unified search across entities

### src/api/challenge/

- `index.ts` тАФ Challenge namespace aggregation
- `comments.ts` тАФ Challenge comment queries and mutations
- `explore.ts` тАФ Challenge discovery (search, featured, browse)
- `favorites.ts` тАФ Favorite/unfavorite mutations
- `likes.ts` тАФ Like/unlike mutations
- `single.ts` тАФ Single challenge CRUD, clone, archive, rebuild

### src/api/task/

- `index.ts` тАФ Task namespace aggregation
- `comments.ts` тАФ Task comment queries and mutations
- `multiple.ts` тАФ Batch task operations (markers, bounds queries)
- `single.ts` тАФ Single task CRUD, status updates

### src/api/taskBundle/

- `index.ts` тАФ Task bundle namespace aggregation
- `queries.ts` тАФ Bundle CRUD and queries

### src/api/user/

- `index.ts` тАФ User namespace aggregation
- `admin.ts` тАФ Admin user management
- `auth.ts` тАФ Authentication (OAuth, whoami, session)
- `notifications.ts` тАФ Notification mutations (mark read, delete)
- `profile.ts` тАФ User profile and settings
