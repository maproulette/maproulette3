## src/api/

API hooks organized by domain. Each `index.ts` aggregates sub-modules into a namespace object.

- `index.ts` вЂ” Top-level API namespace (`api.user`, `api.task`, `api.challenge`, etc.)
- `osm.ts` вЂ” OpenStreetMap API (changesets, elements, Overpass)
- `project.ts` вЂ” Project CRUD, listing, stats
- `search.ts` вЂ” Unified search across entities

### src/api/challenge/

- `index.ts` вЂ” Challenge namespace aggregation
- `comments.ts` вЂ” Challenge comment queries and mutations
- `explore.ts` вЂ” Challenge discovery (search, featured, browse)
- `favorites.ts` вЂ” Favorite/unfavorite mutations
- `likes.ts` вЂ” Like/unlike mutations
- `single.ts` вЂ” Single challenge CRUD, clone, archive, rebuild

### src/api/task/

- `index.ts` вЂ” Task namespace aggregation
- `comments.ts` вЂ” Task comment queries and mutations
- `multiple.ts` вЂ” Batch task operations (markers, bounds queries)
- `single.ts` вЂ” Single task CRUD, status updates

### src/api/taskBundle/

- `index.ts` вЂ” Task bundle namespace aggregation
- `queries.ts` вЂ” Bundle CRUD and queries

### src/api/user/

- `index.ts` вЂ” User namespace aggregation
- `admin.ts` вЂ” Admin user management
- `auth.ts` вЂ” Authentication (OAuth, whoami, session)
- `notifications.ts` вЂ” Notification mutations (mark read, delete)
- `profile.ts` вЂ” User profile and settings
